# Breathing AI

A computer vision project that uses your webcam to detect and classify breathing phases in real time.

YOLOv8 tracks your body from the side, optical flow measures how much your shoulder, chest, and belly move between frames, and a neural network classifies those movement signals into four breathing states: **inhale**, **exhale**, **hold in**, and **hold out**.

---

## How It Works

```
Webcam -> YOLOv8 finds shoulder + hip -> 3 boxes placed on shoulder / chest / belly
       -> optical flow measures x-direction movement inside each box
       -> scipy smooths the 3 signals
       -> data normalized + sliced into 60-frame windows (~3 seconds)
       -> Conv1D + LSTM neural network trained on those windows
       -> live predictions from webcam
```

## Why Side View + Optical Flow?

Earlier versions tracked face and shoulder landmarks from the front using MediaPipe. The signal was weak — the chest barely moves in the Y direction when sitting at a desk.

Switching to a **side view** with **optical flow boxes** gave a much stronger signal:

| Approach | inhale-exhale gap |
|---|---|
| Front view landmarks (MediaPipe) | ~0.001 |
| Side view optical flow (YOLOv8) | ~0.105 |

Optical flow measures how pixels actually move between frames — no landmarks needed for the signal itself. YOLOv8 is only used to position the 3 boxes on the right body area automatically.

---

## CSV Data Format

Each session file contains one row per camera frame:

| Column | Description |
|--------|-------------|
| `frame` | Frame number |
| `shoulder_flow` | Mean x-direction optical flow in the shoulder box |
| `chest_flow` | Mean x-direction optical flow in the chest box |
| `belly_flow` | Mean x-direction optical flow in the belly box |
| `timestamp` | Seconds elapsed since recording started |
| `label` | `inhale` / `exhale` / `hold_in` / `hold_out` |

Positive flow = body expanding forward (inhale direction).
Negative flow = body contracting (exhale direction).
Near zero = no movement (hold phases).

---

## Processed Dataset Format

`data/processed/dataset.npz` is a compressed numpy file:

| Array | Shape | Description |
|---|---|---|
| `X` | `(num_windows, 60, 3)` | Training windows, each 60 frames x 3 features |
| `y` | `(num_windows,)` | Label: 0=inhale, 1=exhale, 2=hold_in, 3=hold_out |

Generated by running `process_data.py`. Re-run it any time you add new sessions.

---

## Model Architecture

- **Input:** 60-frame sliding window, 3 features per frame
- **Conv1D(3 -> 32, kernel=5)** — detects local patterns (rising/falling signal)
- **MaxPool1D** — downsamples from 60 to 30 time steps
- **LSTM(32 -> 64)** — learns patterns across the sequence over time
- **Dropout(0.3)** — prevents overfitting
- **Dense(64 -> 4)** — outputs one score per class
- **Softmax** — converts scores to probabilities

> **Note:** Always use `py -3.12` to run scripts, not just `python` or `py`.

## Recording Data

```bash
py -3.12 scripts/collect_data.py
```

- Set `VIEW = 'side'` at the top of the script (default)
- Sit **sideways** to the camera — your shoulder and hip must both be visible
- Three colored boxes appear on your torso — orange (shoulder), cyan (chest), green (belly)
- Press a key when each breathing phase starts — the label stays until you press the next key
- **W** = inhale &nbsp; **S** = exhale &nbsp; **E** = hold_in &nbsp; **D** = hold_out
- Press **Q** to stop — data is saved automatically to `data/side/session_XXX.csv`

**Tips:**
- Camera at waist/chest height, slightly to your side
- Breathe deliberately so the chest and belly clearly expand
- Aim for 10-15 full breathing cycles per session (~2-3 minutes)

---

## Processing Data

Run after recording new sessions to regenerate the training dataset:

```bash
py -3.12 scripts/process_data.py
```

Reads all CSVs from `data/side/`, smooths the flow signals, normalizes per-session, slices into 60-frame windows, saves `data/processed/dataset.npz`.

---

## Training the Model

```bash
py -3.12 scripts/train_model.py
```

Trains the Conv1D + LSTM model and saves the best weights to `models/breathing_model.pt`.
Key settings in the script: `EPOCHS`, `BATCH_SIZE`, `LR` (learning rate).

---

## Live Prediction

```bash
py -3.12 scripts/predict_live.py
```

- Opens your webcam and runs the trained model in real time
- Sit sideways — same position as during recording
- Top-right: current predicted label + confidence (e.g. `exhale 91%`)
- Left side: confidence bars for all 4 classes
- Waits ~3 seconds on startup to fill the 60-frame buffer before predicting
- If you move out of frame, the buffer resets automatically
- Press **Q** to quit
